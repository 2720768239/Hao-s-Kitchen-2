# 极饿时代 UI 设计说明文档

版本：V1 当前实现口径  
更新：2026-06-03

## 1. 当前设计边界

本文件描述当前 V1 的有效设计口径。

以下内容已经确认：

- 前台是 invite 驱动的单页面
- 首页 `/` 固定展示 `群雄归隐`
- V1 不做 AI 推荐
- V1 不使用辣椒图标、辣椒贴纸、辣椒角标、辣椒印章等视觉元素
- 提交成功不做独立成功页，只在当前单页给出 `懂你意思` 的轻提示
- 后台按移动端主厨工具台设计

说明：

- `docs/ui风格.png`、`docs/image-01.png` 等历史图片可以保留作为风格参考
- 但其中的 AI 区块和辣椒视觉元素不是当前 V1 的实现依据

## 2. 整体视觉方向

关键词：

- 海报感
- 纸张肌理
- 黑 / 红 / 米白
- 江湖饭局气质
- 强主厨人格

设计原则：

- 不做电商感
- 不做 PC 中后台灰面板
- 不做辣椒主题视觉
- 不为了海报感牺牲状态可读性

## 3. 前台设计

### 3.1 页面结构

有效 invite 页面由以下区域组成：

1. 开场区
2. 菜单区
3. 底部固定操作区
4. 提交弹层
5. 成功提示

### 3.2 开场区

必须包含：

- 主标题 `极饿时代`
- 固定副标 `全员饿人`
- 状态标识 `英雄集结`
- 开场说明
- 向下看菜 / 到底部报名的引导

### 3.3 菜单区

菜卡包含：

- 菜图
- 菜名
- 短描述
- 标签
- 操作按钮
- 状态提示
- 已点人信息

状态表达：

- `available`：可选
- `holding`：显示 `有人先盯上了`
- `claimed`：显示 `已被 XX 拿下`
- `unavailable`：显示 `暂不出战`

### 3.4 底部固定操作区

固定条包含：

- 左侧清单入口
- 辅助文案 `馋好了就报上名来`
- 右侧主按钮 `馋`

展开态固定分为两组：

- `你已经馋上的`
- `已有馋主`

### 3.5 提交弹层

形式：

- 底部弹层或抽屉

字段：

- 名字
- 备注

### 3.6 提交成功反馈

当前 V1 的成功反馈为：

- 关闭提交弹层
- 在当前单页顶部或内容区显示短提示：`懂你意思`

不再要求：

- 独立成功页
- `你饿完了 / 你馋完了 / 我去厨房了` 的单独全屏反馈

### 3.7 关闭态

以下情况统一展示 `群雄归隐`：

- 首页 `/`
- token 缺失
- token 不存在
- 饭局已归档

## 4. 主厨端设计

### 4.1 页面范围

当前 V1 只覆盖：

- 登录页
- 首页
- 待做清单
- 菜单管理
- 编辑菜品
- 历史饭局
- 历史详情

### 4.2 首页

首页重点是：

- 营业状态卡
- `英雄集结 / 群雄归隐`
- 邀请链接
- 复制链接
- 轻提醒

当前 V1 不要求单独的“今晚概览”信息卡。

### 4.3 待做清单

列表优先展示：

- 菜名
- 点菜人
- 备注
- 提交时间

### 4.4 菜单管理

列表项包含：

- 菜名
- 标签
- 上架状态
- 编辑入口
- 排序位置
- 上移 / 下移操作

### 4.5 历史饭局

列表展示：

- 归档时间
- 单数
- 菜数

详情展示：

- 创建时间
- 归档时间
- 每笔提交的点菜人
- 备注
- 提交时间
- 菜品清单

## 5. 组件规范

### 5.1 状态标签

统一用于：

- `英雄集结`
- `群雄归隐`
- `有人先盯上了`
- `已被 XX 拿下`
- `暂不出战`

### 5.2 主按钮

- 固定文案：`馋`
- 不能做成默认蓝色按钮样式

### 5.3 成功提示

- 短、明确、有主厨语气
- 当前定稿：`懂你意思`
- 不做系统 toast 以外的额外跳转

## 6. 当前设计验收

满足以下条件即可视为与当前 V1 对齐：

1. `/` 只显示 `群雄归隐`
2. invite 页面有营业中的开场区
3. invite 页面在同一页完成看菜、占位、提交
4. 提交后只出现 `懂你意思` 的轻提示
5. 主厨端按当前收缩版页面范围提供能力
6. 页面中不存在 AI 推荐区块
7. 页面中不存在辣椒视觉元素
