# 2026-06-02 Figma 极饿时代 UI 设计对话纪要

## 基本信息

- 日期：2026-06-02
- 工具：Figma
- 主题：极饿时代 UI 设计阶段推进与高保真修正
- 目标：根据 UI 设计说明和参考图，完成 Figma 设计稿，并保证后续可用于代码实现

## 对话目标

本轮对话的核心目标是把 `极饿时代 UI Design V1` 的 Figma 文件补齐到可交付状态，并逐步逼近参考高保真设计图，同时避免把页面做成不可实现的“截图稿”。

重点约束包括：

- 前台要保留强烈海报感和世界观
- 后台要做成移动端主厨工具台
- 不能使用辣椒图形、辣椒贴纸、辣椒角标或辣椒印章
- 设计成果必须能用于代码实现，而不是整屏位图覆盖

## 主要流程

### 1. 确认 Figma 文件与当前页面

- 确认了目标文件是 `极饿时代 UI Design V1`
- 读取了 Figma 页面结构
- 发现文件中已有前台高保真页面，包含前台若干状态画板和视觉规范页
- 同时确认文件里存在本地颜色变量，但没有可复用的组件实例或完整设计系统资源

### 2. 初步补齐页面

在最初的推进中，补了以下内容：

- 前台提交成功态
- 后台移动端 4 个页面
- 组件规范页

当时的目标是先把交付结构补全：

- `主厨首页`
- `待做清单`
- `菜单管理`
- `编辑菜品`
- `提交成功态`
- `组件规范`

### 3. 第一轮高保真优化

为了尽量接近参考图，做过一轮高保真修正，包括：

- 使用参考图裁切出前台各状态的视觉层
- 通过整屏图片覆盖提高相似度
- 调整人物、印章、贴纸、墨刷背景、纸张噪点等视觉元素

但随后用户指出一个关键问题：

- 这种做法更像“把图片贴上去”，而不是可用于代码实现的 Figma 图
- 用户需要的是基于原始设计图，用 Figma 原生图层画出来，便于后续导出代码

这一步之后，方案被纠正。

### 4. 结构性重做

根据用户反馈，重新调整思路：

- 移除整屏位图覆层
- 保留原生可编辑结构
- 把人物插画、菜图、头像等作为独立图片资源
- 其余元素全部使用 Figma 原生节点表达

这之后，逐步把前台改成：

- 标题
- 红色宣言横幅
- 人物插画
- 印章
- 贴纸
- 底部引导
- 菜卡
- 抽屉
- 报名态

### 5. 图片资源独立化

为了保证代码可实现性，专门生成并导入了独立图像素材：

- 开场人物插画
- 群雄归隐人物插画
- 5 张独立菜图
- 2 个独立头像素材

这些素材被放在：

- `docs/figma-assets/`

后续 Figma 图层里，图片不再是大范围截图，而是分散在合适的图片节点中。

### 6. 参考图差异的关键解释

用户多次强调当前设计和参考图仍有差距，并要求进一步贴近原稿。

在对话中，已经明确了两者的主要区别：

- 参考图是“整张海报式构图”，不是干净 UI + 人物贴图
- 参考图的红色横幅是刷痕和掉墨，而不是规则矩形
- 参考图的人物、印章、贴纸、墨刷、旧纸噪点是一起构成的整体
- 当前设计需要保留“可编辑图层”，但视觉上要更像一张完整海报

### 7. 最终开场态深化

对开场态进行了多轮针对性修正：

- 放大并重新定位人物
- 调整人物手势和构图
- 用可编辑的刷痕节点模拟红色横幅的粗粝边缘
- 添加黑色墨刷飞白
- 把印章改成双圈旧印
- 给印章加掉墨和旧纸感
- 把胸前标签改成衣服上的黑章

最后，开场态已经从“规整 UI”转向更接近参考图的“海报合成感”。

### 8. 菜单态与抽屉态调整

在菜单态上做了这些修正：

- 菜卡密度更接近原稿
- 菜图改成独立图片资源
- `holding`、`claimed`、`available` 状态更清晰
- claimed 状态右侧改成头像语义
- 底部操作区更接近票据式海报条

清单抽屉也进行了补强：

- 列表上移，避免压住底部操作条
- 每条菜目使用独立图片节点
- 保留可编辑列表结构

### 9. 归隐态调整

归隐态做了类似海报化处理：

- 角色放大
- 墨刷背景保留但弱化
- 印章位置和尺寸修正
- 保持和开场态一致的世界观语汇

## 当前结论

最终形成的方向是：

- 视觉尽量接近参考图
- 但结构必须是可编辑、可导出、可用于代码实现的 Figma 图
- 不能退化成整屏截图稿

当前成果已经满足：

- 前台、后台、规范页都有结构
- 关键状态都有独立图层
- 图片素材独立化
- 可继续进入代码实现阶段

## 后续建议

如果继续推进，下一步应当是：

1. 把 Figma 中最终的前台页面导出为代码实现基准
2. 如有更高一致性需求，补齐更多独立素材，而不是继续贴整屏截图
3. 将当前 Figma 结构作为前端实现依据，避免再回到纯视觉拼贴方案
