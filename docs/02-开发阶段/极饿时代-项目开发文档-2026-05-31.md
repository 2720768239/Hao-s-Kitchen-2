# 极饿世代 项目开发文档

版本：V1 当前实现口径  
更新：2026-06-03

## 1. 技术方向

当前 V1 采用单一 Next.js Web 应用，包含：

- 前台 invite 单页
- 主厨工具台
- SQLite 持久化
- 本地图片上传目录

## 2. 当前页面与路由

### 2.1 前台

- `/`：关闭态
- `/invite/[token]`：营业中的 invite 单页

### 2.2 主厨端

- `/chef/login`
- `/chef`
- `/chef/to-cook`
- `/chef/dishes`
- `/chef/dishes/[id]`
- `/chef/history`
- `/chef/history/[id]`

## 3. 当前主流程

### 3.1 前台

`进入有效 invite -> 看开场区 -> 选择菜品占位 -> 打开底部抽屉核对 -> 填写名字和备注 -> 提交 -> 显示 懂你意思`

### 3.2 主厨端

`登录 -> 开席/打烊 -> 查看待做清单 -> 维护菜单 -> 查看历史饭局`

## 4. 服务端规则

### 4.1 营业状态

- `gathering`：营业中
- `archived`：已归档

规则：

- 没有有效 `gathering` 饭局时，前台统一显示关闭态
- 主厨点击 `英雄集结` 创建下一场饭局和新 invite token
- 主厨点击 `群雄归隐` 归档当前饭局、删除未提交占位、使旧 invite 永久失效

### 4.2 菜品与占位

- 同一道菜同一时间只能有一个有效占位
- 同一道菜在同一饭局中只能被正式提交一次
- 占位有效期固定为 `3 分钟`
- 占位过期后自动释放

### 4.3 提交成功反馈

- 提交成功后不跳独立成功页
- 由前台客户端在当前页给出 `懂你意思` 提示

## 5. 当前数据与存储

### 5.1 数据库表

- `meal_sessions`
- `chef_sessions`
- `dishes`
- `dish_holds`
- `orders`
- `order_dishes`

### 5.2 本地持久化

前台浏览器使用 `localStorage` 保存：

- `clientSessionId`
- 上次填写过的顾客名字

目的：

- 刷新后恢复“我的占位”
- 提高重复提交时的填写效率

## 6. 主厨鉴权方案

当前实现采用：

- 登录接口校验主厨口令
- 登录成功后写入 `chef_session` cookie
- 服务端只保存 session token 的 hash
- 主厨页面和 `/api/chef/*` 接口统一校验 session

未登录时：

- 页面跳转 `/chef/login`
- API 返回 `401`

## 7. API 清单

### 7.1 前台

- `GET /api/public/state/[inviteToken]`
- `GET /api/public/holds?inviteToken=...&clientSessionId=...`
- `POST /api/public/holds`
- `DELETE /api/public/holds/[id]`
- `POST /api/public/orders`
- `GET /api/events/[inviteToken]`

### 7.2 主厨端

- `POST /api/chef/login`
- `POST /api/chef/logout`
- `POST /api/chef/business-status`
- `GET /api/chef/to-cook`
- `GET /api/chef/dishes`
- `POST /api/chef/dishes`
- `PATCH /api/chef/dishes/[id]`
- `POST /api/chef/dishes/reorder`
- `GET /api/chef/history`
- `GET /api/chef/history/[id]`
- `POST /api/chef/uploads`
- `GET /api/chef/events`

## 8. 上传策略

当前上传约束：

- 仅支持 PNG / JPG / WEBP
- 文件大小不超过 5MB
- 上传后通过 `/uploads/[filename]` 访问

持久化目录：

- 默认 `data/uploads`
- 容器内默认 `/app/data/uploads`

## 9. 初始化与 seed 策略

`db:seed` 当前只用于：

- 空库初始化默认菜单

规则：

- 如果库中已经有菜品，`db:seed` 不再覆盖现有菜单
- 容器启动时只执行 `db:migrate`
- 不再在每次容器启动时自动执行 `db:seed`

## 10. 实时刷新

当前实现使用 SSE：

- 前台监听 `/api/events/[inviteToken]`
- 主厨端监听 `/api/chef/events`

收到事件后客户端执行整页刷新，不在浏览器端合并复杂共享状态。

## 12. 部署说明

### 12.1 本地

首次初始化菜单：

```powershell
npm install
npm run db:migrate
npm run db:seed
npm run dev
```

### 12.2 Docker

应用启动时会自动：

```text
npm run db:migrate
npm run start
```

首次部署后，如果数据库为空，需要手动执行一次：

```bash
docker compose exec app npm run db:seed
```

### 12.3 持久化

当前持久化目录：

- SQLite：`data/hao-kitchen.sqlite`
- 上传图片：`data/uploads`

## 13. 备份

保留以下脚本：

- Windows：`scripts/backup.ps1`
- Linux/macOS：`scripts/backup.sh`

备份内容：

- `data/hao-kitchen.sqlite`
- `data/uploads/`
