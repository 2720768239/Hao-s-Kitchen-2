# 极饿时代 image-01 高保真 UI 资产样板规格

日期：2026-06-01  
参考图：`docs/image-01.png`  
基准画布：移动端内容宽度 `375px`

## 1. 目标

基于 `docs/image-01.png` 产出一套可供前端试装的高保真 UI 资产样板，并补充现有 UI 设计说明文档中缺少的可执行规范。

本轮只处理 `image-01.png`。完成样板验收后，再决定是否扩展到 `image-02.png` 与 `image-03.png`。

本轮目标是视觉接近参考图且可用于开发，不承诺从合成展示图中还原逐像素一致的原始素材。

## 2. 交付内容

### 2.1 UI 设计说明补充

在 `docs/极饿时代-UI设计说明文档-2026-05-31.md` 中新增“基于 image-01 的前台高保真规范”，包括：

- 字体栈、字号、字重与行高
- 色板、透明度与使用场景
- 圆角、边框与阴影
- 页面间距、组件间距与安全区
- 以 `375px` 为基准的响应式规则
- CSS、文本、SVG 与 PNG 的职责边界
- 资产索引表

### 2.2 高保真资产包

输出目录为：

```text
docs/ui-assets/image-01/
  generated/
  icons/
  reference-crops/
  README.md
  contact-sheet.png
```

其中：

- `generated/`：生成式补全或重绘的 PNG 资产
- `icons/`：适合代码接入的 SVG 图标
- `reference-crops/`：从参考合成稿中裁切的 PNG 对照素材
- `README.md`：资产索引、用途、尺寸与接入建议
- `contact-sheet.png`：资产总览图

## 3. 资产拆分原则

### 3.1 优先使用 CSS 与文本实现

以下内容不固化为图片：

- 标题、菜名、描述、按钮文案与状态文案
- 容器、卡片、输入框、遮罩与常规边框
- 常规圆角、阴影与间距
- 可通过 CSS 实现的简单色块、线条与贴纸底

这样可以保持响应式能力、可访问性与后续文案调整能力。

### 3.2 使用 SVG 重绘的元素

以下图标统一重绘为 SVG：

- 菜单图标
- 关闭图标
- 删除图标
- 上箭头
- 收起箭头

SVG 图标保持粗线条、简洁、略带印刷感。图标不使用辣椒形象。

### 3.3 使用 PNG 的元素

以下复杂视觉使用 PNG：

- 开场页黑白主厨版画插画
- 归隐页背影主厨版画插画
- 米白旧纸纹理
- 黑色底纹
- 红色油墨泼洒纹理
- 圆形红色印章底图
- 菜品缩略图参考裁切

复杂插画和纹理允许使用生成式补全或重绘，以恢复被设备外壳、文字或其他图层遮挡的区域。

### 3.4 参考裁切与最终资产分离

从合成稿直接裁切的素材只作为对照依据。若裁切区域包含文字、手机外壳、遮挡或压缩噪点，应在资产索引中标明，不直接作为最终生产资产。

## 4. 生成式补全边界

生成式补全与重绘允许用于：

- 恢复插画被遮挡区域
- 扩展纹理以支持平铺或不同屏幕比例
- 生成无文字、透明背景或可组合的视觉资产
- 统一黑红米白、粗粝版画、旧纸印刷风格

生成资产不得：

- 引入辣椒图形或辣椒装饰
- 把文字烘焙进需要响应式变化的视觉素材
- 包含设备外壳、系统状态栏或 Home Indicator
- 引入与参考风格冲突的现代渐变、霓虹色或电商图标

## 5. UI 规范结构

### 5.1 基准画布

- 基准内容宽度：`375px`
- 页面布局以手机竖屏为主
- 底部固定操作区和抽屉必须处理安全区
- 大屏手机限制内容最大宽度，避免无节制拉伸

### 5.2 响应式断点

文档按以下区间整理：

| 区间 | 定位 | 适配策略 |
| --- | --- | --- |
| `320-359px` | 紧凑屏 | 收紧页面边距、卡片内部间距与部分标题字号 |
| `360-389px` | 标准屏 | 以 `375px` 设计基准直接实现 |
| `390-429px` | 宽屏手机 | 适度增加留白，保持卡片与底部操作区比例 |
| `430px+` | 大屏手机 | 内容区域设置最大宽度并居中 |

### 5.3 设计数值

实施阶段根据参考图测量并在 UI 说明文档中给出：

- 字体栈、字号、字重、行高
- HEX、RGB 与透明度
- 圆角、描边宽度、阴影参数
- 页面边距、卡片间距、组件内边距
- 弹层高度、底部固定区高度与安全区策略

由于参考输入是展示合成图而非设计源文件，数值将标注为“建议实现值”，不伪造设计工具中的精确原始参数。

## 6. 页面覆盖范围

样板规范至少覆盖 `image-01.png` 中展示的以下前台状态：

1. 开场态
2. 菜单主视图
3. 底部抽屉展开态
4. 留名提交弹层
5. 群雄归隐态

人物插画、纸张纹理、印章、贴纸与油墨效果优先做成可复用资产，不按页面重复导出。

## 7. 资产索引要求

`docs/ui-assets/image-01/README.md` 对每项资产记录：

| 字段 | 说明 |
| --- | --- |
| 文件名 | 可直接用于前端引用的文件名 |
| 类型 | PNG、SVG 或参考裁切 |
| 用途 | 对应页面和组件 |
| 建议尺寸 | 默认像素尺寸与倍图策略 |
| 背景 | 透明、米白底、黑底或可平铺 |
| 接入方式 | 图片、CSS background、mask 或 inline SVG |
| 状态 | 可直接接入、需视觉确认或仅供参考 |
| 备注 | 遮挡、裁切限制与替换建议 |

## 8. 验收标准

本轮交付满足以下条件时，样板包可进入前端试装：

1. UI 说明文档包含字体、字号、行高、色值、阴影、圆角、间距和响应式断点。
2. `docs/ui-assets/image-01/` 中存在结构清晰的 PNG 与 SVG 资产包。
3. 复杂插画与纹理不包含设备外壳、状态栏和 Home Indicator。
4. 可通过 CSS 与文本实现的内容未被不必要地固化成图片。
5. 简单图标已重绘为 SVG，缩放后保持清晰。
6. 资产索引明确区分“可直接接入”“需视觉确认”“仅供参考”。
7. 资产总览图可用于快速检查风格一致性。
8. 全部视觉资产遵循现有 UI 文档中的禁用项，不出现辣椒形象装饰。

## 9. 后续扩展

`image-01` 样板验收后，可按相同目录结构与规范扩展：

- `docs/ui-assets/image-02/`
- `docs/ui-assets/image-03/`

扩展阶段优先复用已经生成的纹理、插画、印章与 SVG 图标，避免同类资产重复制作。
