> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the first-stage family dining ordering app with shared meal sessions, duplicate prevention, a chef-facing to-cook queue, and real-time sync.

**Architecture:** Use a single web app with a public ordering flow and an admin flow. Shared meal sessions are the core boundary: every diner joins one active session, all selections are written into that session, and duplicate dishes are blocked at the client and server layers. The admin side owns meal-session creation, dish management, and the to-cook queue. Real-time updates should flow from the server to all open clients so shared state stays aligned.

**Tech Stack:** Assumed for planning purposes: Next.js App Router, TypeScript, Prisma, SQLite for local development, PostgreSQL for deployment, and Server-Sent Events or WebSocket for real-time updates.
