# 极饿时代 V1 Cloud Implementation Plan

> Historical plan note: this file is kept as a dated execution artifact. The current V1 scope and behavior are defined by the main docs under `docs/00-需求阶段`, `docs/01-设计阶段`, and `docs/02-开发阶段`.

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build and deploy the complete cloud-hosted V1 family dining app with invite-only public ordering, chef-password administration, SQLite persistence, three-minute holds, SSE refresh notifications, image uploads, history, Docker Compose, and an HTTPS reverse-proxy example.

**Architecture:** Use one Next.js App Router application with route handlers for public and chef APIs. Keep business rules in focused TypeScript services backed by Drizzle and SQLite transactions; use SSE only as a refresh notification channel so clients always re-fetch authoritative state. Persist the SQLite database and uploaded images under `/app/data`, mounted by Docker Compose.

**Tech Stack:** Next.js App Router, TypeScript, React, Drizzle ORM, better-sqlite3, Vitest, Testing Library, Playwright, Docker Compose, Nginx.
