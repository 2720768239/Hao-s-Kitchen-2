# 极饿世代用户使用手册

适用版本：当前 V1 本地/私有部署版本  
更新日期：2026-06-04

## 1. 系统入口

极饿世代分为用户端和主厨后台。

- 用户端：通过主厨后台生成的邀请链接访问，格式为 `/invite/[token]`。
- 主厨后台：访问 `/chef/login`，输入主厨口令后进入。
- 默认首页 `/`：固定显示“群雄归隐”，不作为点菜入口。

默认主厨口令为 `hhhhz`。如果部署环境配置了 `CHEF_PASSWORD_HASH`，实际口令以环境变量对应的 hash 为准。

## 2. 用户端使用方式

### 2.1 进入点菜页面

1. 从主厨处获取邀请链接。
2. 用手机或浏览器打开链接。
3. 如果饭局处于“英雄集结”状态，会看到今晚菜单。
4. 如果饭局已关闭、链接错误或饭局已归档，会看到“群雄归隐”页面。

### 2.2 浏览菜单

菜单以菜品卡片展示，包含菜名、图片、描述和标签。

菜品可能出现以下状态：

- 可选择：可以点击按钮加入自己的嘴馋清单。
- 已临时占位：有人正在选择，暂时不可被其他人提交。
- 已有馋主：已经被其他用户正式提交。
- 已下架：主厨在后台关闭了该菜品，用户不可选择。

### 2.3 临时选择菜品

1. 在想吃的菜品上点击选择按钮，例如“我馋这个”“给我这个”。
2. 选中的菜会进入底部“我馋上的”清单。
3. 临时占位有效期为 3 分钟；超时未提交时，该菜会重新释放。
4. 如需取消，展开底部清单，点击对应菜品旁的“取消”。

### 2.4 提交点菜

1. 选择至少一道菜。
2. 点击底部圆形“馋”按钮。
3. 在弹窗中填写名字。
4. 可选填写备注，备注最多 100 字。
5. 点击“报上名来”完成提交。

提交成功后：

- 已提交的菜会成为“已有馋主”状态。
- 后台“待做清单”会实时出现该订单。
- 当前浏览器会记住填写过的名字，便于下次使用。

### 2.5 常见用户端问题

- 看不到菜单：饭局可能未开启，或邀请链接不是当前有效链接。
- 菜品无法选择：可能已被他人占位、已被提交、已下架，或饭局已关闭。
- 提交失败：可能临时占位已过期，重新选择后再提交。
- 选择后不提交：3 分钟后占位会自动释放。

## 3. 主厨后台使用方式

### 3.1 登录后台

1. 打开 `/chef/login`。
2. 输入主厨口令 `hhhhz`。
3. 点击“进入主厨工具台”。

如果部署环境设置了自定义 `CHEF_PASSWORD_HASH`，请使用对应口令登录。

### 3.2 后台导航

后台底部有四个入口：

- 首页：管理饭局营业状态并复制邀请链接。
- 待做清单：查看当前饭局已提交的菜。
- 菜单管理：新增、编辑、上下架和排序菜品。
- 历史：查看已归档饭局。

### 3.3 开启饭局

1. 进入 `/chef` 首页。
2. 点击“英雄集结”。
3. 系统会创建一场新的饭局，并生成邀请链接。
4. 点击邀请链接旁的复制按钮，将完整链接发给用户。

注意：

- 同一时间只会有一场进行中的饭局。
- 已有进行中饭局时，“英雄集结”按钮不可重复开启。
- 用户必须通过当前邀请链接进入点菜。

### 3.4 关闭并归档饭局

1. 进入 `/chef` 首页。
2. 点击“群雄归隐”。
3. 当前饭局会被关闭并永久归档。

关闭后：

- 当前邀请链接不再可用。
- 未提交的临时占位会被清空。
- 已提交订单会进入历史饭局。
- 关闭的饭局不会重新打开；需要下一场时重新点击“英雄集结”。

### 3.5 查看待做清单

进入 `/chef/to-cook`。

待做清单按订单提交时间和菜品排序展示：

- 菜名
- 用户名字
- 用户备注
- 提交时间

当用户提交点菜后，该页面会自动刷新。当前版本没有“已完成”勾选功能，待做清单表示当前饭局所有已提交菜品。

### 3.6 新增菜品

1. 进入 `/chef/dishes`。
2. 在“新增菜品”表单中先上传菜品图片。
3. 填写菜名，最多 20 字。
4. 填写描述，最多 100 字。
5. 填写标签，可用中文逗号或英文逗号分隔，例如 `下饭，招牌`。
6. 点击“新增到菜单”。

图片要求：

- 支持 PNG、JPG、WEBP。
- 单张图片不超过 5MB。
- 新增菜品默认上架。

### 3.7 编辑菜品

1. 进入 `/chef/dishes`。
2. 在菜品列表中点击“编辑”。
3. 可修改菜名、图片、描述、标签和是否上架。
4. 点击“保存”。

上下架规则：

- 上架菜品会在用户端显示并可被选择。
- 下架菜品会在用户端显示为不可选状态。
- 如果饭局正在进行，菜品变更会推送给用户端刷新。

### 3.8 调整菜单排序

1. 进入 `/chef/dishes`。
2. 在菜品列表中点击“上移”或“下移”。
3. 排序会影响用户端菜单展示顺序，也会影响待做清单中同一订单内菜品的排列。

### 3.9 查看历史饭局

1. 进入 `/chef/history`。
2. 列表展示已归档饭局，包括归档时间、订单数和菜品数。
3. 点击任意历史饭局查看详情。

历史详情包含：

- 饭局创建时间
- 饭局归档时间
- 每个用户的提交时间、备注和菜品清单

## 4. 管理注意事项

- 开饭前先检查菜单是否完整、图片是否可显示、菜品是否上架。
- 发邀请链接前确认首页状态为“英雄集结中”。
- 饭局结束后及时点击“群雄归隐”，避免用户继续提交。
- 若需要更换主厨口令，推荐生成新的 `CHEF_PASSWORD_HASH` 并更新部署环境变量。
- 当前版本没有多账号、多角色和订单完成状态，后台仅面向主厨本人使用。

## 5. 故障处理

### 5.1 主厨口令不对

先确认当前部署是否设置了 `CHEF_PASSWORD_HASH`。

- 未设置环境变量：使用默认口令 `hhhhz`。
- 已设置环境变量：使用该 hash 对应的自定义口令。

### 5.2 用户打开链接后显示群雄归隐

可能原因：

- 饭局未开启。
- 邀请链接来自已归档饭局。
- 链接 token 被复制错误。

处理方式：主厨进入 `/chef`，点击“英雄集结”，复制最新邀请链接后重新发送。

### 5.3 用户提交时提示不可提交

可能原因：

- 临时占位超过 3 分钟。
- 菜品已被其他用户提交。
- 饭局已关闭。

处理方式：用户刷新页面，重新选择当前可选菜品后提交。

### 5.4 菜品图片上传失败

检查图片格式和大小：

- 格式必须是 PNG、JPG 或 WEBP。
- 文件大小不能超过 5MB。

### 5.5 后台数据没有更新

系统使用实时刷新连接。网络异常时可能延迟显示，刷新页面即可重新获取最新状态。
