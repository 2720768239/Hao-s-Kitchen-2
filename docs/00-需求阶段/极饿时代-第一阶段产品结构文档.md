# 极饿时代 第一阶段产品结构文档

更新：2026-06-03

本文档只描述当前 V1 真实结构，不再保留早期多路由草案。

## 1. 当前产品结构

### 1.1 前台

前台只有两类页面：

- `/`：统一关闭态，展示 `群雄归隐`
- `/invite/[token]`：有效 invite 的营业中单页

其中 `/invite/[token]` 单页包含：

1. 开场区
2. 菜单区
3. 底部固定操作区
4. 提交弹层
5. 提交成功提示

### 1.2 主厨端

主厨端只保留：

- `/chef/login`
- `/chef`
- `/chef/to-cook`
- `/chef/dishes`
- `/chef/dishes/[id]`
- `/chef/history`
- `/chef/history/[id]`

不再单独提供：

- 谁点了什么页
- 聚餐会话管理页
- 状态回看页
- 菜品详情页

## 2. 信息结构

### 2.1 前台单页

- 开场海报区：标题、状态、引导
- 菜卡区：图片、名称、描述、标签、状态、操作
- 底部抽屉：我的占位 + 已有馋主
- 提交弹层：名字 + 备注
- 成功反馈：`懂你意思`

### 2.2 主厨工具台

- 首页：营业状态和 invite 链接
- 待做清单：按提交时间看今晚要做什么
- 菜单管理：新增、编辑、上/下架、排序
- 历史饭局：列表和只读详情

## 3. 核心流程

### 3.1 前台

`进入有效 invite -> 看开场区 -> 选菜占位 -> 展开底部抽屉确认 -> 填名字和备注 -> 提交 -> 提示 懂你意思`

### 3.2 主厨端

`登录 -> 英雄集结/群雄归隐 -> 查看待做清单 -> 维护菜单 -> 查看历史饭局`

## 4. 当前确认规则

1. 前台采用单页，不拆多路由流程
2. 首页 `/` 固定关闭态
3. 只有有效 invite 才展示营业中的前台开场区
4. V1 不做 AI 推荐
5. V1 不做辣椒视觉元素
6. 提交成功不跳转成功页，只显示 `懂你意思`
7. 主厨端按当前收缩范围实现，不补“谁点了什么页”和“聚餐会话管理页”
