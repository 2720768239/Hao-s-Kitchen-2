# 极饿世代 完整需求文档

版本：V1 当前实现口径  
更新：2026-06-08

## 1. 产品定义

`极饿世代` 是一个面向家庭聚餐场景的共享点菜工具。

它不是外卖系统、餐厅点单系统或电商购物车。  
它服务的是“朋友来家里吃饭，大家一起决定今晚做什么”的场景。

V1 的核心目标只有三件事：

1. 客人能快速看菜、选菜、报名字
2. 所有人都能看到哪些菜已经被别人盯上或拿下
3. 主厨能立刻知道今晚要做什么，并维护菜单

## 2. 版本边界

### 2.1 V1 包含

- 前台单页 invite 点菜页
- 首页/无效链接统一关闭态
- 主厨登录页
- 主厨工具台首页
- 待做清单
- 菜单管理
- 历史饭局列表与只读详情
- 营业状态开关
- 3 分钟临时占位
- 图片上传
- SQLite 持久化

### 2.2 V1 不包含

- AI 推荐
- 支付
- 注册/普通用户登录
- 多饭局并行
- 多主厨协作
- 谁点了什么独立页面
- 聚餐会话管理独立页面
- 制作状态流转（例如“开始做”“已完成”）

## 3. 角色

### 3.1 来吃饭的人

- 通过有效邀请链接进入点菜页
- 浏览菜单
- 选择菜品
- 看到别人已点或已占位的菜
- 填写名字和备注并提交

限制：

- 不能删除别人占位的菜
- 不能重复正式点成同一道菜

### 3.2 主厨

- 使用口令登录主厨工具台
- 切换 `英雄集结 / 群雄归隐`
- 查看待做清单
- 新增、编辑、上/下架、排序菜品
- 查看历史饭局

## 4. 营业状态规则

系统只有两种营业状态：

- `英雄集结`
- `群雄归隐`

### 4.1 英雄集结

- 当前没有进行中饭局时，主厨点击 `英雄集结` 创建新饭局
- 生成独立、不可猜测的邀请链接
- 客人可通过邀请链接进入营业中的前台单页

### 4.2 群雄归隐

- 主厨点击 `群雄归隐` 后，当前饭局归档
- 旧邀请链接立即永久失效
- 当前未提交占位会被清理
- 前台回到关闭态

### 4.3 前台访问规则

- 首页 `/` 固定展示关闭态，当前为全屏 `饿疯了` 海报图
- 缺少 invite token、token 不存在或饭局已归档时，统一展示同一关闭态
- 只有 `/invite/[token]` 在 token 有效时展示营业中的前台单页

## 5. 前台规则

### 5.1 页面结构

前台是一个单页面流程，包含：

1. 开场区
2. 菜单区
3. 底部固定操作区
4. 提交弹层
5. 提交成功轻提示

关闭态单独采用全屏海报页，不复用营业中页面的区块拼装结构。

### 5.2 开场区

营业中的 invite 页面必须展示：

- `极饿世代` 主标题
- 开场文案
- 当前状态 `英雄集结`
- 向下看菜和到底部报名的引导

菜单区当前不再强制展示独立区块标题和说明文案。

### 5.3 菜品状态

每道菜有四种前台展示状态：

1. `available`
2. `holding`
3. `claimed`
4. `unavailable`

其中核心业务三态仍然是：

- `available`
- `holding`
- `claimed`

`unavailable` 用于主厨下架后前台展示“暂不可选”。

### 5.4 临时占位

- 用户把菜加入底部区时先形成临时占位
- 临时占位时长固定为 `3 分钟`
- 超时自动释放
- 同一道菜同一时刻只能有一个有效占位

### 5.5 正式提交

用户正式点成一道菜，必须同时满足：

1. 已经形成自己的有效占位
2. 点击主按钮 `馋`
3. 填写名字和备注
4. 提交成功

提交成功后：

- 占位转为正式点菜记录
- 不跳转成功页
- 在当前单页里给出轻提示：`懂你意思`

### 5.6 底部固定操作区

底部固定操作区合并以下能力：

- 当前已点信息
- 我自己的占位信息
- 提交入口

展开后固定分组为：

- `你已经馋上的`
- `已有馋主`

规则：

- 只能移除自己的占位
- 别人的占位和正式已点只能查看

## 6. 主厨端规则

### 6.1 页面范围

V1 主厨端只保留以下页面：

- `/chef/login`
- `/chef`
- `/chef/to-cook`
- `/chef/dishes`
- `/chef/dishes/[id]`
- `/chef/history`
- `/chef/history/[id]`

### 6.2 鉴权

- 主厨端页面和 `/api/chef/*` 接口都受口令保护
- 登录成功后写入基于 cookie 的 session
- 未登录访问主厨页面时跳转登录页
- 未登录访问主厨接口时返回 401

### 6.3 首页

主厨首页当前包含：

- 营业状态卡
- 当前有效邀请链接
- 复制邀请链接
- 操作提醒

首页不再要求单独的“今晚概览”区块。

### 6.4 待做清单

至少展示：

- 菜名
- 点菜人
- 备注
- 提交时间

默认按提交时间排序。

### 6.5 菜单管理

支持：

- 新增菜品
- 编辑菜品
- 图片上传
- 描述和标签维护
- 上架 / 下架
- 排序调整

### 6.6 历史饭局

历史饭局提供：

- 列表页
- 只读详情页

详情页至少展示：

- 饭局创建时间
- 饭局归档时间
- 每笔提交的点菜人
- 备注
- 提交时间
- 菜品清单

## 7. 数据与一致性

V1 必须保证：

1. 同一道菜不能被两个人正式提交成功
2. 占位过期可靠释放
3. 打烊后拒绝新占位和新提交
4. 主厨上下架、排序、营业状态变化能同步到前台

## 8. 非功能要求

- Web / H5 优先
- 手机浏览器优先
- 后台为移动端主厨工具台
- 前台和后台都不使用辣椒图标、辣椒贴纸、辣椒角标、辣椒印章等视觉元素
- V1 不出现 AI 推荐区块

## 9. 验收标准

满足以下条件时，V1 可视为完成：

1. `/` 和无效 invite 统一显示关闭态，当前视觉为 `饿疯了` 海报图
2. 有效 invite 在同一页面内完成看菜、占位、提交
3. invite 页展示营业中的开场区
4. 提交成功后给出 `懂你意思` 提示
5. 同一道菜不会被重复正式点成
6. 占位会在 3 分钟后释放
7. 底部展开区正确区分 `你已经馋上的 / 已有馋主`
8. 主厨后台受口令保护
9. 菜单管理支持新增、编辑、上/下架、排序
10. 历史饭局可只读查看
